# Техническое задание для проекта CPA

## Главное:

- Необходимо реализовать web-приложение по макету: [Figma](https://www.figma.com/design/U9mPOGl2XWFWVq5Uttc1JK/CPA-F?node-id=4134-4449&t=b7MonnoULUrJ9Sev-1)
- На данный момент 2 страницы с разными языками
- Десктопная версия работает в режиме **постраничного перелистывания**, мобильная в целях оптимизации не содержит таких переходов и анимаций
- Каждая страница в десктопном режиме занимает 100% высоты, контент **адекватно** умещается в ней
- Все анимации расположены на странице "Animations". Для воспроизведения воспользуйтесь кнопкой _"Present"_ в правом верхнем углу
- Часть данных для постоения страниц приходит от API (см. раздел по API)
- Любые анимации, в том числе и заднего фона, должны работать плавно и во всех современных браузерах ([Пример необходимого покрытия](https://browsersl.ist/#q=baseline+widely+available))

---

### Стилистика

- Pixel Perfect в расположении элементов не требуется, так как проект расчитывается для отображения в самых разных условиях. Обязательно протестируйте устройства с диагональю 13 дюймов
- При этом размеры самих текстов и элементов должны совпадать с макетом
- В проекте используется фиксированное ограничение по ширине (контейнер)
- Анимация на заднем фоне воспроизводится постоянно. На втором экране затемняется оверлеем
- Реализация "блюра" при тригере модального окна обязательна

### Анимации

- Все анимации должны быть выполнены в соответсвии с макетом. **Исключение**: анимация заднего фона может быть больше/меньше по размерам и иметь немного другой временной паттерн воспроизведения
- Анимация перехода на следующий/предыдущий блок активируется одним шагом скролла колесика мыши
- **Обязательно** предусмотрите вариант обратной анимации, когда пользователь возвращается на предыдущий блок
- При переходе на страницу с локальной ссылкой пользователь сразу попадает на нужный блок и анимация воспроизводится по макету. Если из этого положения пойти вперед/назад - все должно корректно работать
- Анимация переключающейся строки на первой странице бесконечна
- Анимация бегущей строки на третьей странице бесконечна
- На мобильных устройствах отключаются все анимации, кроме бегущей и переключающейся строки

### Интерфейс

- Состояния кнопок и ссылок можно найти на вкладке Components
- Желательно, чтобы нельзя было выделить картинки со змеями мышкой, но не обязательно
- Форма обратной связи допускает пустое имя. Остальные два поля обязательны
- Методы для контакта: **Telegram, Whatsapp, Email**
- Окно благодарности не закрывается само по таймеру
- Каждая кнопка на 4м экране тригерит вызов модального окна
- Ссылки в шапке первого экрана - это локальные ссылки на 2, 3 и 4ю страницы по порядку
- На последней странице ссылка "Scroll to top" возвращает пользователя на первый экран
- Ссылки на соцсети пока можно заглушить (#)
- Переход по языковым ссылкам ведет на другой url с контентом на другом языке. Базовая страница (/) - это английская версия, остальные - это пути `/ru` , `/es` и др.
- Переход по локальным ссылкам на другую страницу должен быть доступен. пример: `/es#join`

### API

- Интерактивная документация доступна по адресу [ссылка](https://cpa-server-vtel.onrender.com/api-docs). В ней есть указание различных роутов, заголовков ответа и можно тестировать сам ответ от сервера. Все данные для проекта доступны тут
- Для всех запросов необходимо **обязательно** использовать заголовок `x-api-key` со значением `prodcpakey333`
- Обратите внимание на указание языка в url запроса. Сейчас их два: en и ru. Для получения необходимо подставить локаль, например: https://cpa-server-vtel.onrender.com/ru/benefits Если локаль в url не указана, то по умолчанию ставится en
- Сервер **может работать** в режиме холодного старта. Если не было запросов за прошедшие 20 минут, то первый запрос будет выполняться относительно долго (секунд 30-40). К релизу поведение может быть изменено
  P.S. Можно использовать в сочетании с прелоадером

#### Остальные требования

- Оценка в Lighthouse на мобильных и десктоп устройствах не должна быть меньше 75 в Perfomance и 90 в остальных категориях  
  _P.A. Анимации сильно съедают Perfomance, но в оригинальном проекте удалось сделать 82/76_
- В итоговом коде убрать комментарии и выводы в консоль
