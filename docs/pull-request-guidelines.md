# Pull Request Rules
 Набор правил перед созданием PR.

## 1. Прогнать линтер, отформатировать импорты командой pnpm lint:fix

## 2. Делать маленькие и логичные PR
- Один PR — одна задача
- Крупные изменения разбивать на части

## 3. Понятное описание
Указать:
- Что сделано
- Зачем

## 4. Удалить временный мусор
- Нет `console.log`, закомментированного кода
- Нет временных файлов

## 5. Самостоятельный мини‑ревью
- Понятные названия переменных
- Нет дублирования
- Стиль соответствует проекту
